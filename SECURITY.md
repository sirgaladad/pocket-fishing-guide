# Security Policy — Pocket Fishing Guide

Thank you for helping keep **Pocket Fishing Guide** and its users safe.

## Supported Versions
This project is currently in active development. Security fixes are applied to the **default branch**:

- **Supported:** `main`
- **Not supported:** old forks / old copies / other branches

## Reporting a Vulnerability (Preferred)
Please use **GitHub Private Vulnerability Reporting** so we can fix issues before details become public:

1. Go to the repo **Security** tab
2. Click **Report a vulnerability**
3. Provide as much detail as you can (see “What to include” below)

## Alternate Private Reporting (Email)
If you cannot use GitHub Private Vulnerability Reporting, email:

**security@talewatersandtides.com**

Please include the repository name (**sirgaladad/pocket-fishing-guide**) in the subject line.

**Do not include vulnerability details in a public GitHub issue.**

## What to Include
To help us reproduce and fix the problem quickly, include:

- A clear description of the issue and potential impact
- Steps to reproduce (proof-of-concept if possible)
- Affected pages/paths (URLs) and/or files
- Any logs, screenshots, or error messages (redact sensitive info)
- Your best guess at severity (low/medium/high) if you have one

## Response Timeline (Best Effort)
As a small project, we’ll do our best to respond quickly:

- **Initial response:** within **7 days**
- **Fix / mitigation target:** within **30 days** (depending on complexity)

## Disclosure Policy
We ask that you keep vulnerability details private until a fix is released or we agree on a disclosure timeline together.

Thank you again.
